= SWAR Character Class Warp
:sectnums:

== Overview
SWAR (SIMD Within A Register) Character Class Warp is a technique to accelerate two distinct phases of regex execution:
1. **Search Pre-filter (SearchWarp)**: Rapidly finding the next possible match start position.
2. **Character Class Skip (CCWarp)**: Fast-forwarding through continuous matches in self-looping states (e.g., `[a-z]+`).

== Design Specification

=== 1. State ID and Structure Extension
To support both types of SWAR Warp:
* **CCWarp**: Defined per-state in the transition table using the `CCWarpFlag`.
* **SearchWarp**: Defined per-pattern in the `Regexp` structure, used when no constant SIMD prefix is available.

[source]
----
 32-bit Packed State ID:
 [31] TaggedStateFlag
 [30] AnchorVerifyFlag
 [29] AcceptingStateFlag
 [28] CCWarpFlag
 [27-22] AnchorMask (6-bit EmptyOp)
 [21] WarpStateFlag (UTF-8 skip)
 [20] Reserved
 [19-0] State Index
----

=== 2. Search Pre-filter (SearchWarp)
When a pattern lacks a constant prefix (e.g., `[0-9]+`), the engine uses SearchWarp to skip non-starting bytes.

* **Intelligent Thresholding**: SearchWarp is only enabled if the "Start Set" (possible first bytes of a match) contains **fewer than 64 unique ASCII bytes**. This prevents the overhead of SWAR from outweighing its benefits in broad patterns like `.*`.
* **Infinite Loop Prevention**: Runtime loops MUST verify that the index `i` has actually advanced before issuing a `continue`. If no progress is made, the engine falls back to standard 1-byte DFA transitions.

=== 3. Character Class Skip (CCWarp)
During the DFA optimization phase, a state $S$ is marked for `CCWarp` if it meets the following criteria:

1. **Self-Loop Existence**: There exists an ASCII byte set $C \subseteq [0, 127]$ such that $\delta(S, c) = S \ (\forall c \in C)$.
2. **Purity Guarantee**:
    * The self-loop transition contains no `RecapEntry` (Tags).
    * The self-loop transition requires no `AnchorVerifyFlag` checks.
3. **Determinism**: $C$ matches one of the predefined SWAR kernels (e.g., a single range `[0-9]` or "any-except-newline" for `.*`).

=== 4. Runtime: Hybrid Warp Strategy
Execution loops attempt skipping in the following order of priority. Note that the kernel selection (`switch`) is performed **outside** the inner warp loop to maximize instruction throughput.

[source,go]
----
for i < numBytes {
    // Priority 1: CCWarp (Match continuation)
    if state & CCWarpFlag != 0 && i+8 <= len(b) {
        info := ccWarps[sidx]
        switch info.Kernel {
        case ir.CCWarpSingleRange:
            low, high := info.Low, info.High
            for i+8 <= len(b) {
                v := binary.LittleEndian.Uint64(b[i:])
                if ((v + 0x7f... - high) | (v - low)) & 0x80... != 0 { break }
                i += 8
            }
        // ... (Monomorphized loops for other kernels)
        }
        continue
    }

    // Priority 2: SearchWarp (Match start searching)
    if !anchorStart && (state&ir.StateIDMask) == (searchState&ir.StateIDMask) && len(re.prefix) == 0 {
        if searchWarp.Enabled && i+8 <= len(b) {
            // SWAR skip until a byte from the Start Set is found
            // (Same switch-outside-loop optimization applies)
            if i > oldI { continue }
        }
    }

    // Priority 3: UTF-8 Warp (1-character skip)
    if state & WarpStateFlag != 0 {
        i += 1 + ir.GetTrailingByteCount(b[i])
        continue
    }

    // Priority 4: Standard DFA Transition (1-byte)
    state = table[(state << 8) | int(b[i])]
    i++
}
----

=== 5. SWAR Kernel Variants
* **Kernel 1: Single Range** (`[a-z]`, `[0-9]`)
    * Logic: `((v + 0x7f... - high) | (v - low)) & 0x80... == 0`
* **Kernel 2: ASCII Bitmask** (Complex ASCII sets like `[a-zA-Z0-9_]`)
    * Logic: Verifies all bytes < 128 and checks against a 128-bit bitset.
* **Kernel 3: Any Except Newline** (Optimized for `.*`)
    * Logic: Detects `\n` (0x0A) using zero-byte detection.

== Interaction with UTF-8 Warp
* **Complementary**: SWAR Warp handles wide "surfaces" of ASCII data, while UTF-8 Warp handles "points" of multi-byte character boundaries.
* **Safe Fallback**: If SWAR detects an out-of-range byte, the engine immediately falls back to UTF-8 Warp or standard DFA transitions, ensuring 100% correctness.
* **Performance**: Provides a significant throughput boost for ASCII-heavy payloads, breaking the physical limit of 1-byte-per-transition DFA scanning.
